# Changelog v1.0 - A股财报预期对比分析工具

## 一、遇到的问题

### 问题一：API数据准确性问题

**描述：**
本Skill涉及其他平台的API使用，而且相关API的返回值不太准确（有一些像是停止维护），所以导致AI没有办法第一次就准确使用正确的API（有时候AI觉得有返回值就是对的，其实数据很老等等）。

**具体表现：**
- 最初使用的`stock_yjbb_em`接口返回的数据最新只到2026-01-31，大部分是2021年的历史数据
- AI误以为有返回值就是正确的，没有验证数据的时效性
- 用户指出"今天有公司发财报，比如珠江股份"，但实际脚本返回"未找到"

**解决方案：**
需要将API调试的任务拆解，同AI一起验证正确的API有哪些，并形成独立的python脚本，便于用python命令来验证。

**实际解决过程：**
1. 通过逐一测试akshare的多个接口，发现`stock_notice_report`可以获取实时公告数据
2. 从公告中筛选包含"年报"关键词的条目，准确获取当日发布财报的公司
3. 验证了珠江股份（600684）确实在2026-04-02发布了年报

### 问题三：模型执行差异问题

**描述：**
将Skill放到~/.claude/skills/下后，还是存在击中但是不运行脚本的情况。

**测试表现：**
- glm-5：不执行
- minimax-2.5：不执行
- kimi-k2.5：执行

**参考：**
此表现和[晓天衡宇的指令遵循表单表现一致](https://skylenage.net/sla/leaderboard/detail?key=LLM_BOARD&modalTime=2026-02-27&name=%25E5%25A4%25A7%25E8%25AF%25AD%25E8%25A8%2580)

**分析：**
不同模型对skill description的理解和执行能力存在差异，可能需要：
1. 优化SKILL.md的描述，使其更加明确
2. 添加更多的触发词
3. 考虑在description中增加强制执行的指令

## 二、反思

### 当前版本局限

目前的版本，只支持年报分析，而且运行python脚本就可以实现需求，与是否用AI关系不大。

### 下一步规划

看本skill是否能拓展到：
1. **季报分析**：支持一季报、半年报、三季报的分析
2. **自动化定时**：每天早上7点自动获取并分析
3. **推送通知**：将分析结果推送到邮件或微信
4. **历史对比**：对比历史同期的预期准确率

### 技术债务

1. 需要建立更完善的API测试机制
2. 需要添加数据缓存，避免重复请求
3. 需要处理API限流和错误重试
